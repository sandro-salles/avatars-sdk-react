# Avatar Trivia — Backend RPC + Client Events

An AI avatar hosts a trivia game where questions come from a **backend database** via RPC tool calls, and the UI is driven by **client events** over the LiveKit data channel.

## What it demonstrates

- **Backend RPC** (`@runwayml/avatars-node-rpc`) — the avatar calls `lookup_trivia` to fetch questions from a server-side database instead of making them up
- **Client events** (`@runwayml/avatars-react`) — the avatar fires `next_step` to advance the game UI
- **Mixed tool types** — both `backend_rpc` and `client_event` tools in the same session

## Tools

| Tool | Type | What it does |
|------|------|-------------|
| `lookup_trivia` | `backend_rpc` | Fetches a random question from the server database; returns the question, options, and correct answer to the model |
| `next_step` | `client_event` | Advances the game — delivers the next question plus the previous answer result, updated score, and sound cue |

## How it works

1. The model calls `lookup_trivia` (backend RPC) → your server returns a question from the database
2. The model calls `next_step` (client event) → the browser displays the question, result, and updated score
3. The user answers verbally
4. Repeat from step 1

The key difference from the [client-events example](../nextjs-client-events/): questions are sourced from your backend, not generated by the model. In a real app, `lookup_trivia` could query a database, call an API, or run any server-side logic.

## Quick start

```bash
npx degit runwayml/avatars-sdk-react/examples/nextjs-rpc my-rpc-trivia-app
cd my-rpc-trivia-app
cp .env.example .env.local
# Add your RUNWAYML_API_SECRET and NEXT_PUBLIC_AVATAR_ID

npm install
npm run dev
```

## Architecture

```
lib/trivia-database.ts        (fake question database — server only)
lib/avatar-tools.ts            (tool definitions — shared by server & client)
lib/trivia-personality.ts      (model personality — server only)
      │
      ├── app/api/.../route.ts     (creates session + starts RPC handler)
      └── app/page.tsx             (client: useClientEvents for UI)
```
